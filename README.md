# Critical Evaluation and Reproduction of ML-Based Website Phishing Detection

Final project for **Data Science in Cybersecurity** (Dr. Uri Itai).

This project reproduces and critically evaluates the article
**"Towards Benchmark Datasets for Machine Learning Based Website Phishing Detection: An Experimental Study"**
by Abdelhakim Hannousse and Salima Yahiouche. The article builds a reproducible
benchmark dataset for website phishing detection, extracts 87 features organized
into three groups (URL-based, content-based, and external-service-based), and
compares several classifiers, concluding that Random Forest with hybrid
(all-group) features performs best.

The goal here is not just to reproduce the results, but to test whether the
authors' claims hold up — and where they break down.

## Links

- **Selected article:** [Towards Benchmark Datasets for Machine Learning Based Website Phishing Detection (arXiv 2010.12847)](https://arxiv.org/pdf/2010.12847)
- **Dataset source:** [Web page phishing detection — Mendeley Data (c2gw7fy2j4, v3)](https://data.mendeley.com/datasets/c2gw7fy2j4/3)

The dataset (Dataset B in the article) contains **11,430 websites** — 5,715
legitimate and 5,715 phishing — each described by 87 extracted features plus a
URL identifier and a binary label.

## Execution Instructions

1. **Requirements:** Python 3.9+ with:

   ```bash
   pip install pandas numpy matplotlib scikit-learn xgboost jupyter
   ```

2. **Data:** the dataset is already included in `phishing-detection-project/data/`.
   If only the zip is present, the notebook's loader extracts it and renames
   `dataset_B_05_2020.csv` to `dataset.csv` automatically.

3. **Run the notebook** top to bottom (paths are relative to the `notebooks/` folder):

   ```bash
   cd phishing-detection-project/notebooks
   jupyter notebook phishing_detection_project.ipynb
   ```

4. **Outputs:** all experiment tables are written as CSV files to
   `phishing-detection-project/results/`. Fixed random seeds are used throughout,
   so results are reproducible.

## What the Notebook Covers

- **Data loading & cleaning** — column normalization, duplicate/constant-feature removal, identifier handling.
- **EDA** — feature distributions, outlier analysis, Spearman correlation and redundancy analysis, class balance.
- **Model training** — six models: Logistic Regression, Decision Tree, Random Forest, SVM, Naive Bayes, and XGBoost (added as an extension beyond the article).
- **Evaluation** — accuracy, precision, recall, F1, MCC, ROC-AUC, PR-AUC, confusion matrices.
- **Article-aligned experiments** — feature-group comparison (Experiment I, extended with pairwise group combinations) and feature-class model combination via AND / OR / majority voting / stacking (Experiment II).
- **Critical extensions** — feature selection, redundancy removal, removal of external-service ("suspicious") features, realistic class-imbalance simulation, cross-validation, and error analysis.

## Key Findings

| Model | Accuracy | F1 | MCC | ROC-AUC |
|---|---|---|---|---|
| XGBoost | 96.68% | 0.9668 | 0.9335 | 0.9942 |
| Random Forest | 96.24% | 0.9625 | 0.9248 | 0.9934 |
| SVM | 95.41% | 0.9539 | 0.9082 | 0.9883 |
| Logistic Regression | 93.61% | 0.9359 | 0.8723 | 0.9814 |
| Decision Tree | 93.57% | 0.9357 | 0.8714 | 0.9357 |
| Naive Bayes | 74.28% | 0.7728 | 0.5035 | 0.8051 |

- **Random Forest is the best model among those the article tested** — the claim holds within the article's model set, but XGBoost (not tested by the authors) beats it on every metric.
- **Hybrid features win**, as the article claims, but external-service features alone (just 7 of 81) come within ~2.6 accuracy points of the full model — a stronger "lightweight" option than URL-only features.
- **External-service features dominate**: `google_index`, `page_rank`, and `web_traffic` account for over 35% of Random Forest's total feature importance.
- **The balanced benchmark hides a real problem**: under a realistic ~5% phishing prevalence, accuracy stays at ~96% but precision collapses to ~0.55 — most phishing alerts would be false positives in deployment.

Overall verdict: the article's claims are **mostly supported and reproducible**,
but its model comparison is incomplete and its balanced evaluation setting
overstates real-world performance. Full details are in
[`phishing-detection-project/report/phishing_detection_report.pdf`](phishing-detection-project/report/phishing_detection_report.pdf).
